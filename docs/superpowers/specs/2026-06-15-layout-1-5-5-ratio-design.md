# 界面布局调整为 1:5:5 比例

> **Date:** 2026-06-15
> **Status:** Approved
> **Goal:** 将主窗口三面板布局从 1:3:1 调整为 1:5:5 比例，使界面更美观平衡

## 背景

当前 GDM 主窗口使用 QSplitter 实现三面板布局：
- 左侧：项目面板（文件夹树）
- 中间：缩略图视图
- 右侧：详情面板

当前初始大小设置为 `splitter.setSizes([200, 600, 200])`，即 1:3:1 的比例关系。用户希望调整为 1:5:5 比例，让中间和右侧面板更宽，提升视觉平衡和可用性。

## 设计决策

### 1. 比例值选择

**决定**：使用相对比例值 `[1, 5, 5]` 而非绝对像素值

**理由**：
- Qt 的 QSplitter.setSizes() 会按照给定值的比例分配空间
- 使用相对比例 `[1, 5, 5]` 能自动适应不同窗口大小
- 代码意图更清晰，直接表达 1:5:5 的设计要求
- 用户可随时拖动分隔条覆盖初始比例

### 2. 最小宽度策略

**决定**：不设置面板最小宽度，完全由 QSplitter 和用户拖拽控制

**理由**：
- 用户明确要求不限制最小宽度（选择方案 2）
- QSplitter 默认行为已足够灵活
- 保持简单，避免过度配置

### 3. 修改范围

**决定**：仅修改 `gdm/gui/main_window.py` 第 73-74 行

**理由**：
- 改动极小，仅涉及初始值设置
- 不需要修改 ProjectPanel、ThumbnailView 或 DetailPanel
- QSplitter 会自动处理空间分配和拖拽行为

## 实施计划

### 修改文件
- `gdm/gui/main_window.py` (第 73-74 行)

### 具体改动

**当前代码**（第 73-74 行）：
```python
# 设置初始大小比例 左侧:中间:右侧 = 200:600:200
splitter.setSizes([200, 600, 200])
```

**修改为**：
```python
# 设置初始大小比例 左侧:中间:右侧 = 1:5:5
splitter.setSizes([1, 5, 5])
```

### 验证步骤

1. **运行应用**：启动 GDM，观察初始布局
   - 预期：左侧面板较窄，中间和右侧面板宽度相近且较宽

2. **拖动测试**：鼠标拖动分隔条，验证可自由调整
   - 预期：分隔条可拖动，面板大小实时变化

3. **窗口缩放测试**：调整主窗口大小，观察面板行为
   - 预期：面板按 1:5:5 比例缩放（或保持用户拖拽后的大小）

4. **运行单元测试**：确认无回归
   ```bash
   python -m pytest tests/ -v
   ```

## 影响分析

### 用户体验
- ✅ 右侧详情面板更宽，可显示更多信息
- ✅ 中间缩略图区域与详情面板更平衡
- ⚠️ 左侧项目面板变窄，深层级文件夹名称可能显示不完整（用户可拖拽调整）

### 技术风险
- ✅ 风险极低：仅修改初始值，不改变布局逻辑
- ✅ QSplitter 行为稳定，PySide6 成熟

### 向后兼容
- ✅ 完全兼容：用户设置和配置不受影响
- ✅ QSplitter 会记住用户拖拽后的大小（如有持久化）

## 后续优化建议

如需进一步优化，可考虑：
1. 为左侧面板添加最小宽度（如 100px）避免过窄
2. 在窗口标题或状态栏显示当前比例
3. 添加"重置布局"菜单项，恢复 1:5:5 默认比例
