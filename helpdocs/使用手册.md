# Game Dev Manager 使用手册

## 文件存储位置

程序安装和运行过程中会在以下位置创建和存储文件。

### 程序安装目录

```
C:\Program Files\Game Dev Manager\
  └── GameDevManager.exe          （主程序）
```

**说明**：64 位 Windows 系统上安装在 Program Files 目录中。

### 应用数据目录（%APPDATA%）⭐ 重要

```
C:\Users\{用户名}\AppData\Roaming\Game Dev Manager\
  ├── config.json                  （用户配置文件）
  └── cache/
      ├── cache.db                 （缩略图缓存数据库）
      ├── cache.db-wal             （SQLite 预写日志）
      └── cache.db-shm             （SQLite 共享内存）
```

**用途说明**：
- `config.json` — 保存用户偏好设置（记住上次打开的文件夹、窗口大小等）
- `cache/cache.db` — 缓存系统存放缩略图和图片元数据，二次访问相同目录时快速加载

### 快捷方式和菜单项

```
C:\Users\{用户名}\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Game Dev Manager\
  ├── Game Dev Manager.lnk         （开始菜单快捷方式）
  └── Uninstall Game Dev Manager.lnk （卸载程序快捷方式）

C:\Users\{用户名}\Desktop\
  └── Game Dev Manager.lnk         （桌面快捷方式，如安装时选择创建）
```

### 卸载说明

- **自动删除**：Program Files 中的程序文件、开始菜单快捷方式、桌面快捷方式
- **保留**：`%APPDATA%\Game Dev Manager\` 中的配置和缓存（这些是用户数据）

如需完全卸载（包括清除配置和缓存），请手动删除 `%APPDATA%\Game Dev Manager\` 文件夹。

### 访问应用数据目录

**快速方法**：
1. 在 Windows 文件资源管理器中按 `Ctrl + L`，打开地址栏
2. 输入：`%APPDATA%\Game Dev Manager`
3. 按 Enter，直接跳转到应用数据目录

**或者**：
1. 按 `Win + R`，打开"运行"对话框
2. 输入：`%APPDATA%`
3. 按 Enter，在打开的文件夹中找到 "Game Dev Manager" 文件夹

## 界面布局

主窗口采用三栏式布局，面板间可通过拖拽分隔条调整宽度比例（默认约 1:5:5）。

### 左侧 — 项目面板

文件夹树视图，展示工作区中的目录结构。支持同时添加多个根目录。

- 包含图片的目录以**绿色**标记
- 右键根目录可"从工作区移除"

### 中间 — 缩略图视图

精灵图缩略图网格，以 128x128 图标模式展示选中目录下的所有图片。

- 支持异步加载缩略图，不会卡顿 UI
- 点击图片可在右侧面板查看详情
- 扫描大量图片时显示进度条和状态文字

### 右侧 — 详情面板

选中图片的预览和元数据信息展示区。

- 预览图等比例缩放显示（最大 256x256）
- 元数据字段：文件名、尺寸、文件大小、格式、色彩模式、路径
- 选中多张图片时显示汇总信息（数量、总文件大小）
- 未选中图片时显示空状态

### 底部 — 功能栏

位于菜单栏下方的快捷操作栏，会根据当前悬停的菜单动态切换显示对应的功能按钮。

- 悬停"文件"菜单时显示：打开文件夹、保存工作区、退出
- 悬停"工具"菜单时显示：批量重命名、全量解压、清空缩略图缓存
- 悬停"帮助"菜单时显示：使用手册、欢迎、软件信息

## 打开文件方式

1. **菜单栏**：点击"文件"→"打开文件夹"，选择包含精灵图的目录
2. **拖拽**：将文件夹拖拽到程序窗口（待实现）
3. **命令行**：`python -m gdm.main <文件夹路径>`

## 自动刷新机制

程序会监控当前打开的目录，当目录中的文件发生变化时（新增、删除、修改），会自动刷新缩略图列表。

## 支持的图片格式

- PNG
- JPG / JPEG
- GIF
- BMP
- WebP

## 全量解压

扫描当前选定目录及其子目录中的所有压缩包，一键批量解压。

### 支持的压缩格式

- `.zip`、`.tar`、`.gz`、`.bz2`、`.xz`、`.tgz`

### 使用步骤

1. 在项目面板中选中要扫描的目录
2. 点击"工具"→"全量解压"
3. 确认解压（会显示压缩包数量和总大小）
4. 解压完成后自动刷新当前目录

**注意**：解压后原始压缩包会被自动删除。

## 批量重命名

支持四种重命名模式：

1. **前缀 + 序号**：如 `sprite_001.png`、`sprite_002.png`
2. **查找替换**：查找指定文本并替换为新文本
3. **正则替换**：使用正则表达式进行高级替换
4. **添加后缀**：在文件名末尾添加指定文本

### 使用步骤

1. 在缩略图视图中选中要重命名的图片（支持多选）
2. 点击"工具"→"批量重命名"
3. 选择重命名模式，配置参数
4. 点击"预览"查看重命名效果
5. 确认无误后点击"执行"

## 典型使用场景

### 场景一：快速预览精灵图

1. 启动程序
2. 打开精灵图目录
3. 在中间缩略图网格中浏览所有图片
4. 点击某张图片，右侧详情面板显示预览图和元数据

2. 打开批量重命名对话框

## 日志与调试

### 查看日志

程序运行时的日志输出到控制台。使用 `start.bat` 启动时，控制台窗口会保留，可以查看 print 输出和异常堆栈。

### 常见问题

**问题：缩略图加载失败**
- 原因：图片文件损坏或格式不支持
- 解决：检查图片文件是否正常，尝试用其他图片查看器打开

**问题：程序启动报错 `ModuleNotFoundError`**
- 原因：依赖未安装
- 解决：运行 `pip install -r requirements.txt` 安装依赖

**问题：批量重命名后文件丢失**
- 原因：重命名操作会直接修改文件名
- 解决：重命名前建议备份，或使用版本控制（Git）

**问题：配置文件和缓存存放在哪里？**
- 位置：`C:\Users\{用户名}\AppData\Roaming\Game Dev Manager\`
- 访问方法：按 `Ctrl + L`，在地址栏输入 `%APPDATA%\Game Dev Manager` 回车即可
- 包含：config.json（配置）和 cache/（缓存）文件夹

**问题：如何清空缓存？**
- 方法一：点击菜单"工具" → "清空缩略图缓存"，确认后会清空所有已缓存的缩略图和元数据
- 方法二：手动删除 `%APPDATA%\Game Dev Manager\cache\` 文件夹
- 效果：下次访问目录时需要重新扫描生成缩略图（首次会较慢，后续访问会很快）

**问题：为什么第二次打开相同目录很快？**
- 原因：程序使用缓存机制。首次访问时会扫描目录中的所有图片并生成缩略图缓存
- 优势：第二次及以后访问同一目录时，直接从 SQLite 缓存加载，速度提升 99% 以上
- 提示：如果目录中的文件有变化，程序会自动检测并增量更新缓存

**问题：卸载程序时配置会丢失吗？**
- 是的，但可以避免：默认情况下卸载程序不会删除 `%APPDATA%\Game Dev Manager\` 文件夹中的数据
- 如需保留配置和缓存：按卸载程序的说明完成卸载即可，重新安装后配置会恢复
- 如需完全卸载：手动删除 `%APPDATA%\Game Dev Manager\` 文件夹即可

## 技术说明

- 缩略图异步加载，避免 UI 卡顿
- 元数据从图片文件实时读取，保证信息准确
- 支持大尺寸图片的缩放显示

### 缓存系统

- **持久化存储**：使用 SQLite 数据库存储缓存，跨应用重启保持
- **缩略图格式**：128×128 像素 WebP 格式，文件体积小（单张 2-5KB）
- **增量更新**：仅重新处理发生变化的文件，高效更新
- **自动淘汰**：最多缓存 200 个目录，超过时自动淘汰最久未访问的目录
- **性能指标**：首次访问可能需要数秒，第二次访问时 < 200ms 内完成首屏加载，增量更新 < 500ms 完成
